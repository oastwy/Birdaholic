# Birdaholic 功能优先级与实现难度

## 分级原则

- P0：Android/iOS 共用、改动小、马上提升学习体验。
- P1：仍是共用功能，但涉及数据结构或 UI 流程，需要小心回归。
- P2：依赖 API、定位权限、后台任务或在线数据，平台差异更明显。
- P3：大数据包、网站数据库、AI/第三方图片源，适合单独专项推进。

## P0 已优先实现

- 点击鸟图放大查看，支持双指缩放。
- 每个物种学习界面可写“识别笔记”。
- 学习界面可上传自定义鸟图，保存到当前数据包。
- 排序增加“未学优先”。
- 闪卡学习和鸟种列表支持按“目”筛选。
- 主界面增加 14 天打卡日历和连续天数。
- 关于页增加“鸟瘾综合征”介绍、Logo、隐私原则、致谢框架。
- `Species` 模型加入 `order`、`family`、`image_credit`、`audio_credit` 字段，为全国数据包致谢做准备。

## P1 下一步建议

- 四选一选择题模式：建议先在复习模式中增加“选择题”开关，错误选项从当前牌组随机抽取。
- 滑动学习：当前已加入左右切换前后鸟种、上下滑认识/不认识的基础手势，后续需要真机微调手感，避免和滚动冲突。
- 学习界面 UI 二次整理：底部按钮现在功能增多，建议改成两行操作区或底部抽屉。
- 识别笔记列表入口：可在学习详情页集中查看所有笔记。

## P2 平台/API 功能

- eBird 附近鸟种：需要增加定位权限和地理坐标查询。
- Android 需要配置定位权限；iOS 需要 `Info.plist` 定位说明。
- 推荐先做“手动输入经纬度/地点代码”的附近鸟种，再加手机定位。
- eBird API 与 Xeno API 已有设置基础，继续整合到同一个设置页即可。

## P3 数据与媒体专项

- “鸟图生成换成 iNa/iNaturalist”：建议确认是 iNaturalist 图片源，而不是 AI 生成。若是 iNaturalist，需要做 API 搜索、授权信息、作者和许可证展示。
- 全国内置数据包：Claude 正在做时，本仓库应只保留模型和致谢字段框架，避免同时改大数据包。
- 一个物种两张图：建议数据结构从单图 `image` 逐步扩展为 `images: []`，但需兼容旧包。
- birding.today 网站数据库：建议单独生成可托管 JSON/SQLite，不直接塞进 App 仓库。
